The Language and the Web Framework

Python is an interpreted, object-oriented, high-level programming language with dynamic semantics. Its high-level built in data structures, combined with dynamic typing and dynamic binding, make it very attractive for Rapid Application Development, as well as for use as a scripting or glue language to connect existing components together.

Flask is a micro web framework written in Python. It is classified as a microframework because it does not require particular tools or libraries. It has no database abstraction layer, form validation, or any other components where pre-existing third-party libraries provide common functions. However, Flask supports extensions that can add application features as if they were implemented in Flask itself. Extensions exist for object-relational mappers, form validation, upload handling, various open authentication technologies and several common framework related tools.

Project Outline

  • is601-project1/ <- Root of the project
    • .github/workflows/ <- Directory for the Github Actions for the different regions
      • dev.yml <- Dev deployment instructions for Github Actions to follow
      • prod.yml <- Production deployment instructions for GitHub Actions to follow
      • heroku_awake_dev.yml <- Github Action to keep the web dyno awake via HTTP request
      • heroku_awake_prod.yml <- Github Action to keep the web dyno awake via HTTP request
    • app/ <- Directory for webpage files
      • pages/ <- Directory for HTML content
        • templates/ <- Directory for served pages
        • __init__.py <- File that binds endpoints to the HTML files
      • static/ <- Directory for non-HTML content
        • css/ <- Directory for styling file
        • images/ <- Directory for images used on webpage
        • js/ <- Directory for scripting files
      • __init__.py <- Creates and configures an instance of the Flask application
      • run.py <- File that runs the create command
    • tests/ <- Directory for pytest test cases
    • .coveragerc <- Config file to control coverage.py
    • .dockerignore <- Config file with a list of files that docker ignores
    • .gitignore <- Config file with a list of files that git ignores
    • .pylintrc <- Config file to control pylint.py
    • docker-compose.yml <- Orchestration file that contains the configuration to develop locally, it overrides the Dockerfile to run the flask development server instead of running the gunicorn server that is used for hosting on Heroku
    • Docker file <- File used to create an image to run in a container that runs the program
    • heroku.yml <- Docker file for building heroku image
    • README.md <- File for repo details and instructions
    • requirements.txt <- A list of python dependencies that need to be installed before running
