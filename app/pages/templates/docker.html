The Docker Whale

Docker is an open platform for developing, shipping, and running applications. Docker enables you to separate your applications from your infrastructure so you can deliver software quickly.

Docker Image

An image is a read-only template with instructions for creating a Docker container. Often, an image is based on another image, with some additional customization.

Docker Container

Docker provides the ability to package and run an application in a loosely isolated environment called a container. The isolation and security allows you to run many containers simultaneously on a given host. Containers are lightweight and contain everything needed to run the application, so you do not need to rely on what is currently installed on the host. You can easily share containers, and be sure that everyone you share with gets the same container that works in the same way.

Docker Commands

  • docker-compose - command to use the docker-compose.yml orchestration file that contains the configurations to develop locally
    docker-compose up --build -d <- command to build this project and deploy it locally
  • build - creates an image based on the Dockerfile
  • commit - saves the selected container as an image
  • push - sends the image to a docker repo
  • run - starts a container based on a selected image
  • images - lists the images that are on the host
  • ps - lists the containers that are on the host
  • rm - removes the selected containers/images from the host
  • exec - used to execute commands inside of the container
    ex: docker exec -it 'container' /bin/bash <- runs an interactive bash terminal
