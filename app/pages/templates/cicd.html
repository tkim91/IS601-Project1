Heroku: Our CICD Platform

Heroku is a container-based cloud Platform as a Service (PaaS). Developers use Heroku to deploy, manage, and scale modern apps. CICD stands for Continuous Integration Continuous Delivery and can be broken up into two parts.

  • Continuous Integration: A software development practice where members of a team integrate their work frequently, and each integration is verified by an automated build (including test) to detect integration errors as quickly as possible.
  • Continuous Delivery: A software development discipline where you build software in such a way that the software can be released to production at any time.

GitHub Actions

These are deployment instructions that GitHub will take whenever code is pushed or merged. This project has rules setup so that whenever code is merged into the Master branch, it will deploy the application onto the Heroku dev and prod servers.

Development and Testing

During development, automated tests are set up so that GitHub can let the user know if there are any issues that arise with any changes that have been pushed. The test version of the application gets deployed on the heroku dev server to view your changes on a web page before prod deployment.

Deployment

Once the code passes all tests and is merged, the application gets deployed onto the heroku prod server. The instructions in the prod.yml file in the .github/workflows directory pushes the built prod image onto docker hub.
