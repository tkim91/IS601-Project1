Object Oriented Programming

What is Object-oriented Programming?

Object-oriented programming (OOP) is a programming paradigm that is used by nearly every developer at some point in their programming career. It is so popular that it is taught as the standard way of programming at most institutions. OOP structures a software program into simple, reusable pieces of code blueprints called classes, which are used to create individual instances of objects. There are many object-oriented programming languages including JavaScript, C++, Java, and Python.

Applications built using this paradigm break down the intended solution into reusable classes. Classes are user defined data types and have a blueprint for the structure of methods and attributes. You then create objects which are instances of the defined classes with specific data. Attributes of a class are the information that is stored. They are defined in the class template and are filled with data when an instance of that class is created. The methods of a class are the behaviors of the object: returning data, setting/modifying an attribute, etc.

Let's say that we have to create a daycare application for a pre-school. You have to keep track of names, ages, days attending, parent contact information, teachers, and classrooms. It would not be efficient to create unique code for each human and classroom. Instead, we can create a base class for children called child that contains properties that are shared between all children like age, gender, name, parents, etc. The same can be done for employees and classrooms. You would then add unique attributes and methods to each instanced class to represent differences.

Benefits of Object-oriented Programming

  • OOP models complex things as reproducible, simple structures
  • Reusable, OOP objects can be used across programs
  • Allows for class-specific behavior through polymorphism
  • Easier to debug, classes often contain all applicable information to them
  • Secure, protects information through encapsulation

Four Principles of OOP

  • Inheritance - child classes inherit data and behaviors from parent class
  • Encapsulation - containing information in an object, exposing only selected information
  • Abstraction - only exposing high level public methods for accessing an object
  • Polymorphism - many methods can do the same task

There are four pillars of object oriented programming: inheritance, encapsulation, abstraction, polymorphism.

Inheritance

Inheritance allows classes to inherit features of other classes. Put another way, parent classes extend attributes and behaviors to child classes. Inheritance supports reusability. If basic attributes and behaviors are defined in a parent class, child classes can be created extending the functionality of the parent class, and adding additional attributes and behaviors. The benefits of inheritance are programs can create a generic parent class, and then create more specific child classes as needed. This simplifies overall programming, because instead of recreating the structure of the parent class multiple times, child classes automatically gain access to functionalities within their parent class.

Encapsulation

Encapsulation means containing all important information inside an object, and only exposing selected information to the outside world. Attributes and behaviors are defined by the class template. When an object is instantiated from the class, the data and methods are encapsulated in that object. Encapsulation hides the internal software code implementation inside a class, and hides internal data of inside objects.

Encapsulation requires defining some fields as private and some as public.
  • Private interface: methods and properties, accessible from other methods of the same class.
  • Public Interface: methods and properties, accessible also from outside the class.

    Abstraction

    Abstraction means that the user interacts with only selected attributes and methods of an object. Abstraction uses simplified, high level tools, to access a complex object. This is an extension of encapsulation. An example of this would be that you don't need to know how the engine works in order to drive a car.

    Benefits of Abstraction:
  • Simple, high level user interfaces
  • Complex code is hidden
  • Security
  • Easier software maintenance
  • Code updates rarely change abstraction
    •
    Polymorphism

    Polymorphism means designing objects to share behaviors. Using inheritance, you can have objects can override shared parent behaviors, with specific child behaviors. Polymorphism allows the same method to execute different behaviors in two ways: method overriding and method overloading.

    Method overriding is used in runtime polymorphism. In method overriding, a child class can provide a different implementation than its parent class. Method overloading is done during compile time polymorphism. Methods or functions may have the same name, but a different number of parameters passed into the method call. Different results may occur depending on the number of parameters passed in.

    Glossary of terms related to OOP:

    • instantiation - the creation of an object or class
    • object - an instance of a class that is created dynamically
    • class - an abstract blueprint used to create more specific, concrete objects
    • namespace - an identifier for one or more names from other similar names of different objects, groups or the namespace in general
    • constructor - a special method of a class or structure in object-oriented programming that initializes a newly created object of that type
    • fixture - sample data for testing
    • type hint - the method by which we can force functions to accept the desired data type
    • type cast - converting the data type of an instanced object
    • unit test - a comprehensive testing procedure is individually applied to the smallest parts of a software program for fitness or desired operation
    • static - keyword that can be used to declare a member of a type so that it is specific to that type
    • instance method - a method that belongs to instances of a class, not to the class itself
    • instance property - a property that belongs to instances of a class, not to the class itself
    • static method - a method that belongs to a class rather than an instance of a class
    • static property - a property that belongs to a class rather than an instance of a class
    • encapsulation - an object's ability to hide data and behavior that are not necessary to its user
    • inheritance - a mechanism wherein a new class is derived from an existing class
    • polymorphism - the ability of objects of different types to provide a unique interface for different implementations of methods
    • abstraction - the extraction of relevant information from a larger data set in order to simplify a codebase
    • SOLID - an acronym for five design principles intended to make software designs more understandable, flexible, and maintainable. Single-responsibility principle, Open-closed principle, Liskov substitution principle, Interface segregation principle, Dependency inversion principle.
    • Design Patterns - a repeatable solution to a software engineering problem
