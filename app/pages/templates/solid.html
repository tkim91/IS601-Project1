The SOLID Design Principles

What is SOLID?

SOLID is an acronym for five design principles intended to make software designs more understandable, flexible, and maintainable. Single-responsibility principle, Open-closed principle, Liskov substitution principle, Interface segregation principle, Dependency inversion principle.

  • Single-responsibility Principle - every module, class or function in a computer program should have responsibility over a single part of that program's functionality, and it should encapsulate that part.
  • Open-close Principle - software entities (classes, modules, functions, etc.) should be open for extension, but closed for modification"; that is, such an entity can allow its behaviour to be extended without modifying its source code.
  • Liskov Substitution Principle - an object (such as a class) and a sub-object (such as a class that extends the first class) must be interchangeable without breaking the program
  • Interface Segregation Principle - no code should be forced to depend on methods it does not use
  • Dependency inversion Principle - conventional dependency relationships established from high-level, policy-setting modules to low-level, dependency modules are reversed, thus rendering high-level modules independent of the low-level module implementation details
